# 项目介绍

欢迎来到 **Advanced RAG Engine** (高级检索增强生成引擎) 官方文档站！

---

## 🎯 项目概述

针对超长学术/技术文档在边缘设备部署时易发生的**“检索被无关噪声稀释”**以及**“大模型上下文易发生幻觉/OOM”**等行业共性痛点，本项目通过**语义锚点父子替换**、**自适应语义断崖降噪**以及**线程池双通道并发**，打通了从底层文件解析、滑窗去重、精细召回再到大模型对齐的全链路性能瓶颈。

在 12GB 显卡边缘计算环境下，实现高检索精度、极低时延与极致的回答准确性。

---

## 🏗️ 核心算法原理

### 1. 语义锚点父子替换 (Parent-Child Relocation)
*   **痛点**：传统分块破坏上下文连贯性，块过小信息丢失，块过大语义模糊。
*   **方案**：检索时使用 150 词的精细 Child 块计算相似度，召回后自动关联并映射替换为其对应的 800 词 Parent 父块送入大模型。
*   **收益**：兼顾了“极小语义颗粒的高召回率”与“超大语义长语境的连贯性”。

### 2. 自适应语义断崖阻断 (Semantic Cliff Cutoff)
*   **痛点**：传统固定 Top-K 检索模式易夹带低相关噪点，稀释大模型注意力。
*   **方案**：在 CrossEncoder 重排阶段，动态计算相邻召回片段的分值落差。一旦分值落差超过阈值 (1.5) 发生“断崖式下跌”，即时切断后续段落。
*   **收益**：最大程度防止噪声文本污染上下文，降低 LLM 幻觉率，将无用上下文减少达 50%。

### 3. 线程池双通道并发 (Concurrent Search)
*   **痛点**：多路混合检索（Dense / Sparse）串行调度时延迟高，响应慢。
*   **方案**：结合 Python `ThreadPoolExecutor` 并发调用 Chroma 向量（Dense）与 BM25 关键词（Sparse）双检索通道，多路检索时延压缩至最大单路耗时。
