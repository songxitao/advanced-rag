# .claude

> 存放项目迭代手记（handoff）的目录，用于在 AI agent 会话之间传递上下文、状态与决策记录。

该目录不存放可执行代码，而是以 Markdown 文件记录每次 Agent 会话的工作摘要：当前状态、已完成的修改、关键决策、待办事项以及给后续 Agent 的上下文提示（Context for Resuming）。这些手记按时间顺序排列，后一份通常会声明 'Continues from' 引用前一份，形成一条可追溯的 handoff chain。

### `.claude/handoffs/2026-07-02-201606-gnn-pr-refactoring.md`

记录了 GNN PPR 检索引擎的核心重构：废除 Dense 语义边、实现跨文档实体共现连边与自适应 Hub 剪枝，PPR Accuracy 从 5.2 提升至 5.4。

- `rebuild_graph` (function) - 重构后的建图函数：移除同文档/跨文档 Dense 语义边，改用全局反向索引对每个 Chunk 提取 Top-5 局部特征名词做实体共现连边，并引入 n_pids > max(5, int(N*0.2)) 的 Hub 词自适应剔除防稠密保护。
- `run_personalized_pagerank` (function) - PPR 游走检索入口，已废除 edge_threshold 硬截断参数，避免切断有价值的多跳通路。
- `g2[1] < 0.4 * g1[1]` (expression) - 图检索对前两个节点 G1/G2 的过滤断崖阈值（可微调），用于控制图检索召回量。

## Key Concepts

- **Handoff Chain**: 每次会话以 Markdown 文件记录并在头部声明 Continues from / Supersedes，形成可追溯的会话链。后任 Agent 阅读最新手记即可继承上下文，无需翻阅旧 commit。
- **PPR 检索重构**: 废除 Dense 语义边、改用跨文档实体共现连边 + 反向索引 Top-5 局部名词建图，PPR Accuracy 从 5.2 升至 5.4；Heuristic Walk 以 5.9 Accuracy 成为全场最佳。
- **markitdown 进程隔离**: 通过 subprocess 在独立 conda 环境（markitdown-env）中调用 markitdown CLI，避免 Mammoth/lxml 等重型依赖与 RAG 主环境的 PyTorch/Chroma 产生版本冲突。
- **惰性导入测试策略**: tests/test_loader.py 中对 docx 相关测试使用 importorskip，确保主环境未安装 python-docx 时测试不整体崩溃。
