# .agents

> 定义 AI Agent 开发工作区的全局规则与集成实施计划，约束 Agent 读码行为并指导 markitdown 功能落地。

.agents 模块是项目的元数据与工作区规范层，不包含可执行代码，而是为 AI Agent（如 Cursor/Copilot 等 agentic worker）提供两条核心信息：一是全局开发与调试纪律（MPCR 多级精控读码机制、GPU/CPU 设备环境变量约束），二是按步骤分解的可验证实施计划。AGENTS.md 作为顶层规则文件被所有 Agent 会话读取，markitdown-integration.md 是具体 Task 清单，Agent 需逐条勾选验证并运行校验命令。

## Files

### `.agents/AGENTS.md`

工作区全局规则文件，定义 Agent 读码纪律与架构部署规范。

- `MPCR (多级精控读码机制)` (concept) - 三级精控读码策略：一级图谱优先、二级局部精读（≤30行）、三级备用极简（≤50行），超阈值需人工审批，防止 Agent 盲目通读源码。
- `MinerU PDF 服务约束` (rule) - 强制 RAG_DEVICE=cpu、MINERU_MODEL_SOURCE=local，避免 GPU 显存被 PDF 版面分析/OCR 吃满。
- `Advanced RAG 后端约束` (rule) - 强制 RAG_DEVICE=cuda，利用 GPU 秒级批量计算数百父子块的 Dense Embedding。

### `.agents/plans/markitdown-integration.md`

Markitdown 集成实施的逐步任务清单，含禁止事项与逐步验证命令。

- `Task 1: _convert_via_markitdown` (task) - 在 DocumentLoader 中新增私有方法，通过 conda run -n markitdown-env 子进程调用 CLI，返回 Markdown 字符串。
- `Task 2: .docx 分支替换` (task) - 将 python-docx 直读逻辑替换为 _convert_via_markitdown 调用。
- `Task 3: .xlsx/.xls 新增分支` (task) - 在 .srt 分支前插入 Excel 格式分支，复用 markitdown 转换。
- `Task 4: is_markdown 判定扩展` (task) - coordinator.py 中 is_markdown 集合扩展至包含 .docx/.xlsx/.xls。
- `禁止事项 (Don'ts)` (constraint) - 禁止修改 splitter/graph_search/database/app/reranker/embedding；禁止改 PDF/srt/txt 逻辑；禁止在主环境装 markitdown、禁止 shell=True。

## Key Concepts

- **MPCR (Multi-stage Precision Code Reading)**: 三级递进式代码阅读纪律：一级用 codegraph_explore 图谱定位，二级局部精读限 30 行，三级应急限 50 行；超限需人工审批。核心目标是防止 Agent 低效/违规通读源码。
- **CPU/GPU 职责分离**: MinerU PDF 服务跑在 CPU（RAG_DEVICE=cpu）保护显存，Advanced RAG 向量检索跑在 GPU（RAG_DEVICE=cuda）保证 Embedding 批量计算吞吐。环境变量是路由开关。
- **子进程隔离调用**: markitdown 安装在独立 conda 环境 markitdown-env，通过 subprocess.run(['conda','run','-n','markitdown-env','markitdown',...]) 调用，不污染主环境，且禁止 shell=True。
- **受控变更边界**: 本次集成仅允许改动 src/loader.py 与 src/coordinator.py，splitter/graph_search/database/app/reranker/embedding/PDF/srt/txt 全量冻结，确保变更面最小化。

## Internal Relationships

- `.agents/AGENTS.md` → `.agents/plans/markitdown-integration.md`: AGENTS.md 的全局读码纪律（MPCR）约束 markitdown-integration.md 中 Agent 执行每一步时的代码读取行为，且该计划本身被 AGENTS.md 的 '二级精读≤30行' 规则覆盖。
- `.agents/plans/markitdown-integration.md` → `src/loader.py`: 计划直接指导修改 src/loader.py：新增 _convert_via_markitdown 方法、替换 .docx 分支、新增 .xlsx/.xls 分支。
- `.agents/plans/markitdown-integration.md` → `src/coordinator.py`: 计划指导修改 src/coordinator.py 的 is_markdown 判定，使新格式文件被正确归类为 Markdown 类文档进入 RAG 管线。
- `.agents/AGENTS.md` → `src/loader.py`: AGENTS.md 中 '不要修改 splitter/graph_search/database/app/reranker/embedding' 的禁止事项，间接划定 loader.py 和 coordinator.py 为本次集成唯一允许改动的文件。
